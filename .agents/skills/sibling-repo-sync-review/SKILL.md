# 兄弟仓库同步复核

## 工作流程

1. 修改前用 `realpath` 或明确路径确认目标仓库。
2. 阅读每个仓库当前生效的 `AGENTS.md`、本地文档、当前分支/status 和同名文件布局，不假设兄弟目录完全一致。
3. 应用模式前，先并排比较对应文件。
4. 保留各仓库自己的名称、路由、文档、env key、前端包名和验证命令；传播时移除来源项目措辞。
5. 需要共享的契约继续共享，特别是共享 cache/API key；除非用户要求，不要加 repo-specific 前缀。
6. 每个仓库只暂存本轮交付文件，不添加无关本地数据、生成物或历史未跟踪文件。
7. 每个仓库独立验证；一个兄弟仓库通过不代表另一个通过。

## 验证

- 使用 `rg`/`diff` 确认没有本机绝对路径、来源项目名、未完成占位或跨项目描述残留。
- 运行各仓库适用的测试或静态检查。
- 每个触达仓库都运行 `git diff --check`、`git diff --cached --check` 和 `git status --short`。

## 交付证据

交付时说明目标仓库、同步文件、保留的仓库差异、每个仓库运行的验证、暂存/未跟踪状态，以及哪些内容刻意没有传播。
