# 前端签名加密配置说明

## 配置入口

前端安全链路配置样例位于：

```bash
apps/web-antd/.env.security.example
```

本地开发建议复制到 `.env.development.local`，多租户构建建议写入对应 `.env.tenant_xxx.local`，生产环境建议通过 CI/CD 或部署平台注入，避免把真实密钥提交到仓库。

## 必填变量

- `VITE_ADMIN_SECURITY_APP_ID`：当前前端接入的 AppID，需要与后端 `app_id` 或秘钥管理中的 `uuid` 一致。
- `VITE_ADMIN_SIGNATURE_ENABLED`：是否启用请求签名与响应验签。
- `VITE_ADMIN_CRYPTO_ENABLED`：是否启用请求加密与响应解密。
- `VITE_ADMIN_SIGNATURE_TYPE`：签名方式，生产建议 `R`。
- `VITE_ADMIN_CRYPTO_TYPE`：加密方式，当前浏览器端仅支持 `A`。
- `VITE_ADMIN_SECURITY_AES_KEY`：AES-CBC KEY，需要与后端当前 AppID 的 AES KEY 一致。
- `VITE_ADMIN_SECURITY_AES_IV`：AES-CBC IV，需要与后端当前 AppID 的 AES IV 一致。
- `VITE_ADMIN_SIGNATURE_PRIVATE_KEY`：前端请求签名私钥，需要与后端“用户 RSA 公钥”成对。
- `VITE_ADMIN_SIGNATURE_PUBLIC_KEY`：后端响应验签公钥，需要与后端“服务端 RSA 私钥”成对。

## 页面查看

登录后进入 `工具 / 安全调试台`，顶部“前端安全配置”区域会显示当前构建是否已注入 AppID、AES、RSA 等配置。页面只展示“已配置 / 未配置”，不会展示密钥明文。

## 本地访问地址

前端签名、加密和本地 RSA 生成依赖浏览器 Web Crypto。除 `localhost`、`127.0.0.1` 等本机地址外，普通 `http://` 局域网 IP 不属于安全上下文，浏览器会禁用 `crypto.subtle`。本地联调请优先使用 `http://localhost:5666` 或 `http://127.0.0.1:5666`；如果必须通过局域网 IP 访问，请改为 HTTPS。

## 安全约束

Vite 的 `VITE_*` 变量会被打包进浏览器产物，不能把它当作服务端私密存储。生产环境应结合 HTTPS、短时有效签名、防重放、MFA、IP 白名单和后端密钥轮换使用；真实服务端私钥仍只允许保存在后端配置、K8s Secret 或密钥管理系统中。

已提交的 `.env.development` 和 `.env.tenant_*` 只保留非敏感构建参数与配置位置说明，不提交 AES/RSA 明文值。
