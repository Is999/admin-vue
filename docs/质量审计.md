# admin-vue 重构防漂移与质量审计

审计日期：2026-05-11

## 结论

本次前端重构基于 Vben 5.x 工程内核，保留后台运行所需的 Ops、System、Profile、Tools 四类生产路由，删除 Vben 示例路由、Demo 页面、Telegram 无关业务页面和旧外链入口。裁剪后功能入口更聚焦，构建体积和维护面同步下降。

## 功能防漂移

- 运维管理：保留 Collector、任务控制台、任务队列、任务实例、用户标签。
- 系统管理：保留管理员、角色、权限、字典配置、缓存、审计日志、密钥管理。
- 个人中心：保留个人资料、站内消息、MFA 管理。
- 工具页：保留 API 文档和安全调试台。
- 登录安全：登录页固定展示图形验证码，并随登录请求提交 `key / captcha`，与后端强校验保持一致。
- 安全配置：补充 `.env.security.example` 和 `docs/安全配置说明.md`，安全调试台展示当前前端签名、验签、加密、解密配置状态。
- 权限控制：保留 `accessCodes` 驱动的页面、按钮和 API 权限判断。
- 多语言：保留 `zh-CN / en-US`，新增业务消息统一工具 `tBizMessage / tBizDynamic / tBizFallback`。

## 质量提升

- 路由布局：生产路由仅保留 `ops / system / profile / tools` 模块，入口清晰。
- 代码瘦身：相对旧前端，排除依赖和构建产物后文件数从 1148 降到 1055，代码文件从 1060 降到 975。
- 消息统一：`message.success/error/warning/info` 的中文业务提示已统一接入多语言工具。
- 验证码统一：移除登录验证码环境开关，验证码文案统一接入 `zh-CN / en-US` 多语言。
- 加解密配置：前端安全变量集中使用 `VITE_ADMIN_SECURITY_* / VITE_ADMIN_SIGNATURE_*`，页面只展示配置状态，不泄漏密钥明文；提交的 env 文件只保留说明和占位，真实值放入 `.local` 或 CI/CD。
- 追踪统一：请求层统一生成和透传 `X-Trace-Id / X-Timestamp / traceparent / X-Span-Id`，错误提示可显示 trace。
- 错误上下文：请求客户端在保持 `code/message/data` 兼容的同时保留 `response/config/traceId`，页面级 catch 不再丢失排障上下文。
- 安全统一：签名、加密、MFA、密码强度、文件上传下载复用集中工具。
- 风格统一：继续使用 Vben 官方推荐的 monorepo、`apps/web-antd`、`packages`、`internal` 布局。
- 构建优化：生产构建增加 `vendor-antd / vendor-vxe` 稳定分包，入口 `bootstrap` 从约 1067KB 降至约 777KB，首屏入口压力下降约 27%。

## 守护测试

- `apps/web-antd/src/router/routes/__tests__/access-routes.test.ts`：校验生产路由名称唯一、核心入口完整、示例/无关业务路由不会回流。
- `apps/web-antd/src/utils/request/__tests__/trace.test.ts`：校验 trace_id 归一化和 W3C `traceparent` 生成规则。
- `packages/@core/base/shared/src/utils/__tests__/resources.test.ts`：脚本加载测试去除外网 CDN 依赖，避免 CI 受网络波动影响。
- `packages/effects/request/src/request-client/request-client.test.ts`：校验业务错误保留旧字段和响应链路上下文。
- `packages/**/__tests__`：保留 Vben 核心工具、请求、布局、弹窗、偏好设置等基础能力测试。
