# admin-cron-vue 架构说明

> **AI 开发必读**：AI 参与前端开发、重构或接口联调前，必须先阅读 [admin-cron AI开发规范](../../ark-admin-cron/docs/site/角色文档/后端开发/AI开发规范.md)。涉及接口、错误码、安全链路和多语言时，要同步检查后端文档与前端国际化资源。

## 精简范围

当前前端基于 Vben 5.x 工程内核，仅保留：

- `apps/web-antd`: Ant Design Vue 应用。
- `packages`: Vben 核心包和项目依赖包。
- `internal`: Vben 工程配置包。
- `scripts`: 构建、检查和部署脚本。

已删除 Vben 示例应用、mock 后端、playground、演示路由和演示页面，避免生产包引入无关代码。

## 应用目录

- `src/api/common.ts`: 通用接口类型。
- `src/api/core`: 登录、菜单、用户、文件传输基础接口。
- `src/api/cron`: Cron 任务、队列、Collector、用户标签接口。
- `src/api/system`: 管理员、角色、权限、缓存、日志、秘钥等系统接口。
- `src/router/routes/modules`: 静态路由模块。
- `src/constants/permission-codes.ts`: 权限码常量。
- `src/utils/request`: 请求链路工具，统一生成和解析 `X-Trace-Id`。
- `src/utils/security`: 签名验签、加解密、MFA、`X-Trace-Id`。
- `src/views/cron`: Cron 业务页面。
- `src/views/system`: 系统管理页面。
- `src/locales/langs`: 中文和英文语言包。

## 路由与权限

- 页面权限使用 `meta.authority`。
- 按钮权限使用 `asActionPermission`。
- 超级管理员由后端 `isSuperAdmin` 或角色 ID 1 兜底。
- 前端权限码必须与后端 `admin_permission.uuid` 保持一致。

## 多语言

路由标题统一放在：

- `src/locales/langs/zh-CN/cron.json`
- `src/locales/langs/en-US/cron.json`

新增页面时必须补齐中英文 key，不允许把路由标题硬编码在模块中。

## 请求链路

- 所有请求自动携带 `Accept-Language`。
- 所有请求自动携带 `X-Trace-Id`。
- 全局错误提示会拼接后端响应体或响应头中的 `traceId`，便于用户截图后直接定位日志。
- 开启安全策略时，请求会附加 AppID、签名、加密和 MFA 票据。
- 后端响应会按 `{ status, code, message, data, traceId, spanId }` 拆包。


- 登录、强制改密、MFA、权限码、Cron 页面、Collector 页面均可正常访问。
